## 增量更新说明

### 更新检测

当以下情况发生时，需要更新对应的文档：

| 变更类型 | 需更新的文档 |
|----------|--------------|
| 服务层代码变更 | ARCHITECTURE.md, API.md |
| 数据库 Schema 变更 | DATABASE.md, API.md |
| CLI 命令变更 | CLI.md |
| Web 组件/页面变更 | WEB.md |
| 开发流程/规范变更 | DEVELOPMENT.md |
| 项目结构变更 | DEEPWIKI.md, ARCHITECTURE.md |

### 更新流程

```bash
# 1. 获取最新 commit hash
git rev-parse HEAD

# 2. 查看变更文件
git diff --name-only <old-hash>..HEAD

# 3. 根据变更文件确定需要更新的文档

# 4. 更新文档后，更新此文件的版本信息

# 5. 提交文档更新
git add docs/wiki/
git commit -m "docs: 更新 DeepWiki 文档至 <short-hash>"
```

### 变更检测脚本

```bash
#!/bin/bash
# check-wiki-update.sh

LAST_HASH=$(cat docs/wiki/.version 2>/dev/null || echo "")
CURRENT_HASH=$(git rev-parse HEAD)

if [ "$LAST_HASH" != "$CURRENT_HASH" ]; then
  echo "检测到代码变更，需要更新文档"
  echo "上次版本: $LAST_HASH"
  echo "当前版本: $CURRENT_HASH"
  
  # 检测变更类型
  CHANGED_FILES=$(git diff --name-only $LAST_HASH $CURRENT_HASH 2>/dev/null)
  
  echo "$CHANGED_FILES" | grep -q "services/" && echo "→ 需更新: ARCHITECTURE.md, API.md"
  echo "$CHANGED_FILES" | grep -q "database/" && echo "→ 需更新: DATABASE.md"
  echo "$CHANGED_FILES" | grep -q "cli/" && echo "→ 需更新: CLI.md"
  echo "$CHANGED_FILES" | grep -q "web/" && echo "→ 需更新: WEB.md"
fi
```
