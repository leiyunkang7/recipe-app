## 下厨房

### 视觉特征
- 主色调：绿色 (#4CAF50)
- Logo：绿色厨房图标
- 布局：标题大字在顶部，用料和步骤分区展示

### 截图类型

#### 1. 详情页截图
- **标题位置**：顶部大字，通常1-2行
- **作者/浏览量**：标题下方灰色小字
- **用料区**：
  - 有"用料"标签
  - 食材列表清晰，每行一个食材
  - 格式：`食材名 数量` 或 `食材名：数量`
- **步骤区**：
  - 数字编号（1、2、3...）
  - 每步可能配图
  - 步骤说明较详细

#### 2. 步骤页截图
- 单独展示某个步骤
- 大图配简短说明
- 可能有"小贴士"标注

### 识别要点
```
标题 → recipe.title
用料区 → recipe.ingredients
步骤区 → recipe.steps
烹饪时间 → recipe.cookTimeMinutes（如有标注）
```

### 示例解析
```
[截图文字]
番茄炒蛋
by 美食家小王 · 12.3万浏览

用料
番茄 2个
鸡蛋 3个
葱花 适量
盐 少许
油 适量

步骤
1. 番茄洗净切块，鸡蛋打散备用
2. 锅中放油，倒入蛋液炒至凝固盛出
3. 锅中再放少许油，倒入番茄翻炒
4. 番茄出汁后倒入炒蛋，加盐调味
5. 出锅撒葱花即可

[解析结果]
title: "番茄炒蛋"
ingredients:
  - {name: "番茄", amount: 2, unit: "个"}
  - {name: "鸡蛋", amount: 3, unit: "个"}
  - {name: "葱花", amount: 1, unit: "适量"}
  - {name: "盐", amount: 1, unit: "少许"}
  - {name: "油", amount: 1, unit: "适量"}
steps:
  - {stepNumber: 1, instruction: "番茄洗净切块，鸡蛋打散备用"}
  - {stepNumber: 2, instruction: "锅中放油，倒入蛋液炒至凝固盛出"}
  ...
```

## 小红书

### 视觉特征
- 主色调：红色 (#FF2442)
- 布局：图片为主，文字在图片下方或评论区
- 特点：常有emoji，标题吸引眼球

### 截图类型

#### 1. 图文笔记
- **封面图**：菜品成品图，可能有文字叠加
- **正文**：图片下方，包含食材和步骤
- **评论区**：可能有补充说明

#### 2. 多图轮播
- 第一张：成品图+标题
- 中间：食材图/步骤图
- 最后：完成图

### 识别要点
```
标题（常带emoji） → recipe.title（去除emoji）
正文 → 提取食材和步骤
标签 → recipe.tags
```

### 特殊处理
- 小红书笔记常有emoji，需清理
- 食材可能混在正文中，需要识别"食材"、"准备"等关键词
- 步骤可能用 "①②③" 或 "Step1/2/3" 格式

### 示例解析
```
[截图文字]
🍅 番茄炒蛋｜超下饭的家常菜！

食材准备：
番茄 2个
鸡蛋 3个
小葱 1根

做法：
①番茄切块，鸡蛋打散加少许盐
②热锅凉油，蛋液倒入快速滑散盛出
③番茄下锅炒出汁水
④倒入鸡蛋翻炒均匀
⑤撒葱花出锅～

#家常菜 #下饭菜 #快手菜

[解析结果]
title: "番茄炒蛋"  // 去除emoji和副标题
tags: ["家常菜", "下饭菜", "快手菜"]
ingredients:
  - {name: "番茄", amount: 2, unit: "个"}
  - {name: "鸡蛋", amount: 3, unit: "个"}
  - {name: "小葱", amount: 1, unit: "根"}
```

## 美食杰

### 视觉特征
- 主色调：橙色
- 布局：信息密度高，步骤详细
- 特点：每步配图，常有营养信息

### 截图类型

#### 详情页
- **标题区**：大字标题 + 作者信息
- **主料/辅料**：分开列出
- **步骤区**：编号+图片+说明
- **营养信息**：可能有热量、蛋白质等

### 识别要点
```
主料/辅料 → recipe.ingredients（合并）
烹饪时间 → recipe.cookTimeMinutes
难度 → recipe.difficulty
营养信息 → recipe.nutritionInfo
```

### 示例解析
```
[截图文字]
番茄炒蛋

主料：番茄2个、鸡蛋3个
辅料：葱适量、盐适量、油适量

烹饪时间：15分钟  难度：简单

步骤：
1. 准备食材，番茄切块
   [配图]
2. 鸡蛋打散，加少许盐
   [配图]
3. 热锅下油，炒蛋盛出
   [配图]
...

营养信息（每100g）：
热量 89千卡 | 蛋白质 6.2g | 脂肪 5.8g

[解析结果]
title: "番茄炒蛋"
cookTimeMinutes: 15
difficulty: "easy"
nutritionInfo: {calories: 89, protein: 6.2, fat: 5.8}
```

## 通用识别策略

### 1. 先识别App来源
根据以下特征判断：
- Logo/品牌名
- 主色调
- 布局风格
- 字体样式

### 2. 通用提取流程

```
1. 标题识别
   - 通常是最大字号
   - 位置在顶部
   - 可能含emoji（需清理）

2. 食材提取
   - 搜索关键词：用料、食材、主料、辅料、准备、原料
   - 识别格式：
     * "食材名 数量单位" → 番茄 2个
     * "食材名：数量单位" → 番茄：2个
     * "数量单位 食材名" → 2个番茄
   - 处理特殊值：适量、少许、若干 → unit字段

3. 步骤提取
   - 搜索关键词：步骤、做法、方法
   - 识别编号格式：
     * 数字：1. 2. 3. 或 1、2、3、
     * 圆圈：①②③
     * 英文：Step1/2/3
   - 提取说明文字（去除配图描述）

4. 时间提取
   - 搜索关键词：时间、分钟、小时
   - 区分准备时间和烹饪时间

5. 份量提取
   - 搜索关键词：人份、份量、份
   - 提取数字

6. 难度提取
   - 简单/容易/快手 → easy
   - 中等/普通/一般 → medium
   - 困难/复杂/高级 → hard
```

## 信息提取技巧

### 食材解析正则

```
中文数字转换：
一 → 1, 二 → 2, 三 → 3, 四 → 4, 五 → 5
六 → 6, 七 → 7, 八 → 8, 九 → 9, 十 → 10
半 → 0.5

常见格式匹配：
/^(.+?)\s*[:：]?\s*(\d+(?:\.\d+)?)\s*(个|克|g|ml|勺|杯|片|根|颗|瓣|块|条|把|适量|少许)$/u

示例：
"番茄 2个" → {name: "番茄", amount: 2, unit: "个"}
"盐：少许" → {name: "盐", amount: 1, unit: "少许"}
"油 适量" → {name: "油", amount: 1, unit: "适量"}
"生抽 1勺" → {name: "生抽", amount: 1, unit: "勺"}
```

### 步骤清理

```
去除内容：
- 配图标记：[图片]、(图)、配图
- 时间标记（单独）：用时5分钟（应提取到durationMinutes）
- 无意义前缀：如图所示、如下

保留内容：
- 核心操作说明
- 关键提示
```

### 标题清理

```
去除：
- emoji表情
- 副标题分隔符后的内容：｜、|、-、——
- 营销词汇：超简单、必学、0失败、yyds

示例：
"🍅 番茄炒蛋｜超下饭的家常菜！" → "番茄炒蛋"
"【必学】红烧肉-肥而不腻" → "红烧肉"
```

### 分类推断

```
根据标题和食材推断：
- 含"汤"、"羹" → 汤羹
- 含"面"、"饭"、"粥" → 主食
- 含"蛋糕"、"面包"、"饼干" → 烘焙
- 含"凉拌"、"沙拉" → 凉菜
- 含"饮"、"汁"、"茶" → 饮品
- 步骤少于5步且时间短 → 快手菜
- 默认 → 家常菜
```

### 菜系推断

```
根据食材和做法推断：
- 含"辣椒"、"花椒"、"麻辣" → 川菜
- 含"咖喱" → 东南亚
- 含"寿司"、"味噌"、"日式" → 日式
- 含"泡菜"、"韩式" → 韩式
- 含"芝士"、"奶油"、"西式" → 西式
- 默认 → 中式
```

## 多截图合并策略

当用户提供多张截图时：

1. **封面图**：提取标题、封面图URL
2. **食材图**：提取完整食材列表
3. **步骤图**：合并所有步骤，注意顺序
4. **营养图**：提取营养信息

合并规则：
- 标题：取第一个识别到的
- 食材：合并去重（同一食材取更完整的信息）
- 步骤：按顺序合并
- 其他信息：补充缺失字段
